Random Bookmark Opener

Click the toolbar icon to open a random bookmark. Use the filters below to narrow the candidate pool. All fields take comma-separated values and are case-insensitive. Each field has its own match any / match all mode — for example, set Tags to match all to require a bookmark to have every tag you list.

Include filters

Only bookmarks that pass every populated include field are eligible. Leave a field blank to skip it. If all three are blank, every bookmark is eligible.

Exclude filters

Bookmarks matching an exclude rule are dropped, even if they pass the include filters. Default mode is match any (drop on any single match).

Date added

Restrict candidates to bookmarks added on or after the earliest date, and on or before the latest date. Leave either blank for no bound on that side. Both blank = no date filter.

Behavior

When greater than 1, draws that many distinct candidates without replacement. Only the first opened tab steals focus; the rest open in the background. Capped at 50.

Before opening, the chosen page is fetched and its HTML scanned for these phrases (case-insensitive). If any match, a different bookmark is picked instead, drawing without replacement until enough live candidates are found or the pool is exhausted. Leave blank to disable this check entirely (and skip the network fetch).

Tips

  • Tags use Firefox's awesomebar search. Each value you enter is run through bookmarks.search(), the same search the URL bar uses. It matches your real Firefox tags, but also matches the bookmark's title and URL — there's no API to tell the three apart.
  • Match all is the common case. Enter travel, germany with mode match all to require a bookmark to match both terms (in any of title, URL, or tag). Mode match any = either term.
  • Folders: A plain name (Work) matches any folder with that name in the bookmark's ancestry. Use a slash (Toolbar/Work/Docs) to require a specific path fragment.
  • Keywords: Simple substring match against title + URL.
